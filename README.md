# Kevin

**A creativity-routing architecture.**

> Creativity is not generated by relaxing constraints, but by *routing* between
> unexplored solution spaces, wild variation, method transfer, and epistemic
> selection. The human chooses meaning and direction.
>
> *Kreativität entsteht nicht dadurch, dass man Regeln lockert, sondern dadurch,
> dass man freie Variation mit methodischer Übertragung und harter Selektion
> koppelt. Der Mensch wählt Bedeutung und Richtung.*

Kevin is a member of the DESi / AleXiona ecosystem and inherits its core rule:
**LLM for language, rules for logic.** Generation (wild variation, phrasing a
transferred method) is the language layer; routing, scoring, method-matching and
selection are deterministic and replay-stable. Kevin does **not** "be creative" by
turning a temperature dial — it produces creativity structurally.

---

## Why this is different from "structured creativity"

Most creativity tools either (a) tell a model to *be creative* and hope, or (b)
generate freely within preset zones. Kevin separates three things those approaches
usually blur together:

1. **It does not look for the clever answer first. It looks for the *unworked*
   region.** Creativity does not start at "be creative", it starts at *"here is a
   structurally plausible space nobody has exhausted yet."* That is a far better
   seed than brainstorming.

2. **Then the wild brother is let loose.** A free, aggressive, associative
   instance: unusual analogies, absurd combinations, style breaks, distant domains,
   risky hypotheses, *what-ifs*. It is **allowed to spin** — its job is variation,
   not truth.

3. **Then methods are transferred — content-free.** The strongest idea. We do not
   reuse old *solutions*. We extract the abstract *Denkbewegung* — the operator
   sequence — from a domain and apply it to a new field where nobody has tried it:

   | Method | Origin | Content-free shape |
   |---|---|---|
   | limit-case analysis | mathematics | push variables to the extremes; read off the rule between |
   | element → exception → consequence | law | conditions, defeaters, the effect that follows |
   | red flags first | medicine | exclude the unrecoverable before chasing the likely |
   | source by interest | history | discount evidence by its producer's stake |
   | failure-mode analysis | engineering | enumerate how it breaks before how it works |
   | claim-splitting | DESi | split → check each atom → flag conflicts → consolidate |

   This is **methodical** creativity, not merely semantic creativity — and it is
   close to what strong human creativity actually does: carry a method across a
   border nobody had crossed with it.

This separation of **content from method**, plus controlled recombination of
methods *across domain boundaries*, is the whole thesis.

## The pipeline (the "Dirigent")

```
            ┌──────────────────────────────────────────────────────────────┐
 problem ─► │ 1. EXPLORE   find plausible, under-worked solution spaces     │  solution_space.py
            │ 2. ROUTE     select only the under-explored ones (logic)      │
            │ 3. VARY      send the wild brother in (language)              │  wild_brother.py
            │ 4. TRANSFER  discipline the wildest variants with Layer-9     │  method_library.py
            │ 5. SELECT    keep what is coherent / testable / connected     │  selector.py
            └──────────────────────────────────────────────────────────────┘
                                        │
                                        ▼
                        a ranked, justified possibility space
                                        │
                                        ▼
            ┌──────────────────────────────────────────────────────────────┐
            │ HUMAN GATE   you choose: direction · taste · risk · value     │  human_gate.py
            └──────────────────────────────────────────────────────────────┘
```

Kevin stops at the gate **on purpose**. The machine maps, varies, transfers and
filters; the four decisions a machine should not make stay with the human. The
human's choice is recorded — and a process that *worked* is mined by
`extract_method` into a **new content-free method**, so Layer 9 grows from
successful thinking. The loop closes.

### Solution spaces, scored

Each candidate region gets two deterministic scores and one derived signal:

```
opportunity = plausibility × (1 − exploration)
```

`exploration` rises with overlap against the problem's declared
`known_approaches` — so declaring what you have already tried is what lets Kevin
avoid the crowded space and route the wild brother into plausible, unworked
territory.

### Epistemic selection — "not just pretty"

The selector enforces the part creativity tools usually skip. A dazzling variant
with no falsifiable claim and no bridge back to the problem is **rejected**, however
beautiful. Connectivity and testability are weighted *above* novelty on purpose.
The LLM only *reads* a candidate into structured signals; every score and the final
verdict (`promising` / `tentative` / `rejected`) are computed with fixed arithmetic.

## Quickstart

```bash
make install          # pip install -e ".[dev]"  (no API key, no network needed)
make test             # pytest
make lint             # ruff
make demo             # one full creative pass with the routing trace
```

Or directly:

```bash
python -m kevin "how do we make onboarding feel less like paperwork?" \
    --constraint "no extra headcount" \
    --known "checklists" --known "a welcome email" \
    --trace
```

As a library:

```python
from kevin import Kevin, Problem
from kevin.human_gate import build_briefing

kevin = Kevin()
run = kevin.run(Problem(
    statement="how do we make onboarding feel less like paperwork?",
    known_approaches=("checklists", "a welcome email"),
))
print(build_briefing(run).render())
```

## Design invariants (inherited from DESi)

- **Closed enumerations.** Wild moves, affinity tags and verdicts are fixed sets —
  the chaos is enumerable and auditable.
- **Replay-stable routing.** No PRNG; ids are content hashes. The same problem and
  the same language model yield the same *routing* every time. (A real LLM's
  *wording* is not reproducible — and that is exactly the language/logic boundary.)
- **Read-only provenance.** A `CreativeRun` records what happened; it is never
  silently rewritten.
- **Human approval gate.** No verdict is a decision. Direction, taste, risk and
  value are the human's.

## Language layer

Kevin ships with a deterministic `MockLLM` so the whole pipeline, the tests and CI
run offline with zero setup. To wire a real OpenAI-compatible model (DeepSeek /
GPT-4o, matching the rest of the ecosystem), implement the single seam in
`kevin/llm_client.py:get_default_client()` and set `KEVIN_USE_REAL_LLM=1`. **No
engine changes** — the language boundary is the only thing that moves.

## Layout

```
src/kevin/
  models.py          # closed enums + replay-stable dataclasses (pure data)
  llm_client.py      # the language boundary: LLMClient protocol + MockLLM
  solution_space.py  # stage 1-2: explore + route to under-worked regions
  wild_brother.py    # stage 3: free associative variation
  method_library.py  # stage 4: Layer-9 content-free methods + transfer + extraction
  selector.py        # stage 5: deterministic epistemic selection
  human_gate.py      # the briefing; no decision is made here
  orchestrator.py    # Kevin — routes the whole pipeline
  cli.py / __main__  # python -m kevin
tests/               # one suite per engine + a full-pipeline suite
```

## Status

v0.1 — runnable demonstrator of the architecture. Deterministic engines carry the
real logic; the language layer is a mock pending a wired model. Not a product, not
a claim of general creativity: a structural scaffold for routing toward it.
